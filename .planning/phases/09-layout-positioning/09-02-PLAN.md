Replace thread transition with opacity crossfade and enhance pill interactions (preview card auto-flip, pointer arrow, descendant collapse).

  <action>
  Replace the slide transition in ThreadView with an interruptible opacity crossfade:

  1. **Replace transition state:** Remove `isTransitioning` boolean. Add:
     ```typescript
     const [fadeState, setFadeState] = useState<'idle' | 'fading-out' | 'fading-in'>('idle');
     const targetThreadIdRef = useRef(activeThreadId);
     const fadeTimerRef = useRef<number>();
     ```

  2. **Replace the transition useEffect** (currently lines ~313-330) with:
     - On `activeThreadId` change: save scroll position of leaving thread, cancel any in-progress fade via `clearTimeout(fadeTimerRef.current)`, update `targetThreadIdRef.current = activeThreadId`, set `fadeState('fading-out')`
     - After 75ms timeout (half of 150ms total): check if `targetThreadIdRef.current` still matches `activeThreadId`. If yes, update `prevActiveThreadIdRef.current`, restore scroll position of incoming thread BEFORE fade-in (while content is invisible), then `setFadeState('fading-in')`. After another 75ms, `setFadeState('idle')`.
     - If target changed (interruption): skip the fade-in, the next effect invocation handles the new target.
     - Cleanup: clear timeout on unmount.

  3. **Apply CSS classes to content area:**
     - `fading-out`: `opacity-0 transition-opacity duration-[75ms]`
     - `fading-in`: `opacity-100 transition-opacity duration-[75ms]`
     - `idle`: `opacity-100` (no transition class)

  4. **Remove old slide transition classes:**
     - Remove `transition-transform duration-200 ease-out`
     - Remove `translate-x-[-100%]` and `translate-x-0`
     - Remove the `isTransitioning` state variable

  5. **Scroll restoration timing:** Move scroll restore to happen DURING the fade-out (while opacity is 0), before starting fade-in. This prevents visible scroll jump. The existing `requestAnimationFrame` scroll restore in the `activeThreadId` effect should be updated to coordinate with fade timing.

  - New threads start at top (scrollPosition defaults to 0)
  - 150ms total (75ms out + 75ms in)

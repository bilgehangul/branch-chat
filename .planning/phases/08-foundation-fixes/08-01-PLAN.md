1. **useTextSelection.ts**: The hook currently returns viewport-relative `top`/`left`. It must ALSO return absolute coordinates for the bubble position. Add two new fields to `SelectionState`:
   - `absoluteTop: number` — computed as `rect.top - wrapperRect.top + scrollTop` (requires a second ref parameter OR the container already IS the scroll wrapper). Since useTextSelection receives `scrollRef` (the scroll container) and contentWrapperRef is separate, update the hook signature to accept an optional `wrapperRef: RefObject<HTMLElement | null>` as second argument. If provided, compute:
     ```typescript
     const wrapperRect = wrapperRef.current.getBoundingClientRect();
     const absoluteTop = rect.top - wrapperRect.top + scrollTop;
     const absoluteLeft = rawLeft - wrapperRect.left;
     ```
   - Store these in the bubble state. If wrapperRef is not provided, fall back to current viewport coords.

2. **ThreadView.tsx**:
   - Pass `contentWrapperRef` as second argument to `useTextSelection(scrollRef, contentWrapperRef)`.
   - Move the `ActionBubble` JSX from its current position (line 458-471, OUTSIDE `contentWrapperRef`) to INSIDE the `contentWrapperRef` div (after the HighlightOverlay and GutterColumn, still inside the `<div ref={contentWrapperRef}>` block). This is critical — ActionBubble must be a child of the position:relative wrapper to use absolute positioning.
   - Pass `bubble.absoluteTop` and `bubble.absoluteLeft` instead of viewport coords. Bubble flip: if `absoluteTop < 60` (not enough space above within the wrapper), flip the bubble below the selection by removing the negative translateY transform.
   - **Scroll-dismiss effect**: Add a `useEffect` that:
     - Runs when `bubble` changes (non-null means new bubble appeared).
     - Captures `scrollRef.current.scrollTop` as `initialScrollTop` in a ref.
     - Adds a passive scroll event listener to `scrollRef.current`.
     - On each scroll event, computes `delta = Math.abs(scrollRef.current.scrollTop - initialScrollTop)`. If `delta > 100`, calls `clearBubble()` (instant dismiss, no fade).
     - Cleanup removes the listener.
   - **HighlightOverlay persistence**: Do NOT clear the highlight overlay when bubble is dismissed by scroll. The bubble dismisses via `clearBubble()` which sets bubble to null, but the HighlightOverlay should persist. Currently HighlightOverlay renders when `bubble` is truthy (line 390). Change this: store `lastSelectionRects` in a separate `useRef` that persists after bubble clears. Update it when bubble changes. Render HighlightOverlay from `lastSelectionRects` ref. Add a document click listener that clears the ref (and thus the overlay) when user clicks anywhere outside the bubble.
   - **HighlightOverlay annotationType prop**: When rendering HighlightOverlay, pass the `annotationType` prop. For Phase 8, the highlight is for text selection (user hasn't picked an action yet), so pass `annotationType={undefined}` which will use the default blue color. Plan 08-02 will add the annotationType prop to HighlightOverlay's interface — this wiring ensures ThreadView is ready to pass it. The prop: `<HighlightOverlay rects={lastSelectionRects} annotationType={undefined} />`.

When `flipped` is true, position the bubble 8px below the selection bottom (transform: none, top already adjusted). When false (default), use `transform: 'translateY(calc(-100% - 8px))'` (current behavior).

<decisions_made>

- **AI provider**: Gemini 2.0 Flash via `@google/genai` v1.x (NOT `@google/generative-ai` which is EOL Nov 30 2025)
- **Auth**: `@clerk/express` — `clerkMiddleware()` globally first, then custom `requireApiAuth` using `getAuth(req)` returning 401 JSON (NOT `requireAuth()` which redirects)
- **Search**: Tavily (`@tavily/core`) when on Gemini; OpenAI Responses API when on OpenAI
- **Provider abstraction interface**: `streamChat()`, `findSources()`, `simplify()` — UI-need-driven, not SDK-shaped
- **Login**: Optional (unauthenticated users can use the full app); Google OAuth available in addition to email/password
- **Backend host**: Render (not Railway)
- **SSE streaming**: `fetch + ReadableStream` pattern (NOT `EventSource` — can't send Authorization headers)
- **Zustand store**: flat normalized `Record<id, Thread>` shape with immer middleware (not nested tree — avoids cascade re-renders)
- **Annotated text**: fully re-selectable for any action (Go Deeper, Find Sources, Simplify)
- **Wave chain for Phase 1**: 01-01 (wave 1) → 01-02 (wave 2) → 01-03 (wave 3)

</decisions_made>

<context>
This was a full new-project initialization session. The project is DeepDive Chat — a branching AI research interface where users can select text in AI responses, branch into child threads ("Go Deeper"), annotate with sources/rewrites, and navigate back through a visual thread tree. The biggest architectural risks identified in research: text selection + streaming DOM mutation (disable selection during streaming), Zustand nested tree mutations bypassing reactivity (use immer + flat store), ResizeObserver infinite-loop for gutter pill positioning (read/write phases separated), Framer Motion scroll restoration (only reliable in onAnimationComplete). Provider abstraction must be designed from UI feature needs, not SDK shapes.
</context>

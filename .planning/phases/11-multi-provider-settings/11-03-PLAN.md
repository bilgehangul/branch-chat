    - Test: SettingsContext defaults to tier='free', byokProvider=null, byokKeyVerified=false
    - Test: setByokProvider updates provider and resets model/verified state
    - Test: saveByokSettings encrypts and stores key in localStorage
    - Test: On mount with stored key, decrypts and loads into memory
    - Test: clearByokKey resets to free tier and removes localStorage entry
  </behavior>
  <action>
    1. Create frontend/src/utils/cryptoStorage.ts:
       - `APP_SALT = 'contextdive_byok_v1'`
       - `deriveKey(userId: string)`: SHA-256 hash of userId+salt, import as AES-GCM key
       - `encryptApiKey(userId: string, apiKey: string): Promise<string>`: Generate random 12-byte IV, encrypt, combine IV+ciphertext, return base64
       - `decryptApiKey(userId: string, stored: string): Promise<string>`: Decode base64, split IV+ciphertext, decrypt
       - `clearApiKey(userId: string): void`: `localStorage.removeItem('byok_key_' + userId)`
       - Storage key pattern: `byok_key_${userId}`, `byok_settings_${userId}` (for provider/model/tier)

    2. Create frontend/src/contexts/SettingsContext.tsx:
       ```typescript
       interface SettingsState {
         tier: 'free' | 'byok';
         byokProvider: 'gemini' | 'openai' | 'anthropic' | null;
         byokModel: string | null;
         byokApiKey: string | null;       // in-memory only
         byokKeyVerified: boolean;
         searchProvider: 'tavily' | 'openai-search';
         isModalOpen: boolean;
       }
       ```
       - Provider with children pattern (matching AuthProvider)
       - Initialize state with `tier: 'free'`, all byok fields null/false
       - `useEffect` on mount: read encrypted key from localStorage, decrypt via cryptoStorage, set in-memory `byokApiKey`. Also read `byok_settings_${userId}` for provider/model/tier.
       - `saveByokSettings(userId)`: encrypt key, save to localStorage along with provider/model/tier
       - `clearByokKey(userId)`: remove from localStorage, reset state to free tier
       - `setByokProvider(p)`: update provider, reset model to null, reset verified to false
       - `openModal()` / `closeModal()`: toggle isModalOpen
       - Export `SettingsProvider` and `useSettings()`

    3. Create test files:
       - frontend/src/tests/cryptoStorage.test.ts: Test round-trip encrypt/decrypt, different users produce different ciphertext, clearApiKey removes entry. Note: Web Crypto API may need polyfill in test env — use `@testing-library/jest-dom` setup or mock crypto.subtle.
       - frontend/src/tests/settingsContext.test.ts: Render SettingsProvider, test default state, test setByokProvider resets, test clearByokKey resets to free.
  </action>

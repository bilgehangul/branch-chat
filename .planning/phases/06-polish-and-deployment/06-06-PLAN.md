<task type="checkpoint:human-action" gate="blocking">
  <name>Deploy backend to Render</name>
  <what-to-do>
Deploy the backend to Render. Steps:

1. Go to https://render.com and sign in (or create account)
2. Click "New +" → "Web Service"
3. Connect your GitHub account and select the `child_chats_v1` repository
4. Configure:
   - **Name**: deepdive-backend
   - **Root Directory**: backend
   - **Build Command**: npm install && npm run build
   - **Start Command**: node dist/index.js
   - **Instance Type**: Free (sufficient for v1)
5. Add environment variables (from `.env.example`):
   - `CLERK_SECRET_KEY` — your Clerk secret key
   - `GEMINI_API_KEY` — your Gemini API key
   - `TAVILY_API_KEY` — your Tavily API key
   - `AI_PROVIDER` — `gemini`
   - `CLIENT_ORIGIN` — set to `http://localhost:5173` for now (will update after Vercel deploy)
   - `NODE_ENV` — `production`
6. Click "Create Web Service"
7. Wait for the build to complete (2-5 minutes)
8. Copy the service URL (format: `https://deepdive-backend-xxxx.onrender.com`)

Verify: Run `curl https://YOUR-RENDER-URL/health` — should return HTTP 200

Note: Render free tier has a cold-start delay of ~30-60 seconds after 15 minutes of inactivity. This is normal.
  </what-to-do>
  <resume-signal>Paste the Render URL (e.g., `https://deepdive-backend-xxxx.onrender.com`) to continue</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Deploy frontend to Vercel and wire CORS</name>
  <what-to-do>
Deploy the frontend to Vercel and complete the CORS handshake.

**Step 1: Deploy to Vercel**
1. Go to https://vercel.com and sign in (or create account)
2. Click "Add New" → "Project"
3. Import your `child_chats_v1` GitHub repository
4. Configure:
   - **Root Directory**: `frontend` (IMPORTANT — click "Edit" and set this)
   - **Framework Preset**: Vite (auto-detected)
   - **Build Command**: npm run build
   - **Output Directory**: dist
5. Add environment variables:
   - `VITE_CLERK_PUBLISHABLE_KEY` — your Clerk publishable key (pk_test_...)
   - `VITE_API_BASE_URL` — your Render URL from the previous step (e.g., `https://deepdive-backend-xxxx.onrender.com`)
6. Click "Deploy"
7. Wait for build to complete (~2 minutes)
8. Copy the Vercel URL (format: `https://your-app.vercel.app`)

**Step 2: Update backend CORS to allow Vercel origin**
1. Go to your Render service dashboard
2. Open "Environment" tab
3. Update `CLIENT_ORIGIN` to your Vercel URL (e.g., `https://your-app.vercel.app`)
   - Do NOT include a trailing slash
4. Render will auto-redeploy with the new env var

**Step 3: Enable Vercel branch previews**
- In Vercel project settings → Git → ensure "Deploy from main branch" is enabled
- Branch previews are enabled by default

**Step 4: Verify the full stack**
1. Open your Vercel URL in a browser
2. The app should load in dark mode
3. Try sending a chat message — it should stream a response from Gemini
4. If you see a CORS error, double-check that `CLIENT_ORIGIN` in Render exactly matches your Vercel URL (no trailing slash, https://)
  </what-to-do>
  <resume-signal>Type "deployed" when both services are live and the app is functional, or describe any issues encountered</resume-signal>
</task>
